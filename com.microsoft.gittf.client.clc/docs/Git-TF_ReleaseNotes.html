Git-TF: Release Notes

Git-TF V2.0.3

New Features:

  • Enable to use the commit author as the change set owner.

    With this change the 'git tf checkin --deep' command recognizes the following new options:

    1. --keep-author - Use the commit author as the change set owner when checking in deep. The commit author should be known to TFS either by his name or e-mail address. To use this option you should be either a TFS project administrator or have the "Check in other users' changes" permission.
    2. --ignore-author - Use the the current authenticated user as the changeset owner.
    3. --usermap - Specifies an absolute or relative path to a file providing mapping between Git repository commit authors and TFS user identities. To generate a template mapping file, run check in command with --keep-author option in preview mode. (default: ./USERMAP)

Major Bug Fixes:

  • Performance improvement in deep clone/pull/fetch commands
  • Chunked upload implemented for pending changes generated for commits diff to avoid server timeout for long history of changes

Git-TF V2.0.2

New Features:

  • Clone, Fetch, and Pull --mentions option to add references in the commit comments for any work items linked to the corresponding changeset
  • Enable to save Git-TF configuration parameters in the global git config file.

    With this change:

    1. The 'git tf clone' command does not store in the local repository config file 'depth', 'tag', and 'include-metadata' parameters. The user has to configure them explicitly with 'git tf configure' or 'git config' commands.
    2. The 'git tf configure' command puts to the the local repository config file _only_ the parameters explicitly specified by the user.
    3. The user has an ability to specify git-tf configuration parameters in a global .gitconfig file by using commands:

      git config --global git-tf.depth {1|number} - where number is any positive number between 2 and 2147483647.

      git config --global git-tf.tag {true|false}

      git config --global git-tf.include-metadata {true|false}

    In the similar way these parameters could be specified in the --system git configuration.

Git-TF V2.0.0

New Features:

  • Shelve command - shelves any RefSpec in TFS using Git-TF
  • Shelvesets command - views and delete shelvesets that are on the TFS server
  • Unshelve command - creates a stash from a TFS shelveset
  • File and folder rename support when shelving and checking in. Users can also specify the rename mode to use when performing a check-in or a shelve [--renamemode=all|justFiles|none]
  • Gated build support and ability to configure a default gated build definition
  • Chocolatey package
  • Include git commit meta data in the changeset comments when checking in for Git-TF in both --shallow and --deep modes
  • Fetch and Pull --force option to enable scenarios where there are user permission changes in TFS

Major Bug Fixes:

  • Performance improvement when checking in
  • Disable locking during shallow check in to avoid issues around failing to lock the server directory when checking in
  • Issues around clone leaving untracked files in the repository
  • Multiple other bug fixes

Git-TF V1.0.1

New Features:

  • The ability to specify a check-in comment using --message|-m=Message
  • Checkin --preview|-p option to the check in command
  • All commits that map to a TFS changeset are tagged with TFS_C[ChangesetNumber] tag in git
  • Creates a remote tracking branch to track changesets bridged with TFS
  • Enabled Git-TF to work with bare repositories

Major Bug Fixes:

  • Issues with configure / check in scenario when --shallow is specified
  • Issues with git-tf launcher script when cmd.exe is not in the path
  • Multiple other bug fixes
